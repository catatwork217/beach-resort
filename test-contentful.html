Contentful Connection Test

Testing Your Contentful Setup

This page helps you verify if your Contentful data is accessible.

Step 1: Check Your Contentful Space

Go to: https://app.contentful.com/spaces/buu2xhmfxqzb

Step 2: Verify Content Types

You should have these content types:

  • home - For homepage content (title, description, heroImage)
  • room - For room listings (title, slug, price, description, images)

Step 3: Upload Images

In the Media section, upload your room images and link them to room entries.

Step 4: Expected Data Structure

// Home entry should look like:
{
  "title": "Welcome to Beach Resort",
  "description": "Luxury beachfront accommodations",
  "heroImage": {
    "fields": {
      "file": {
        "url": "//images.ctfassets.net/.../image.jpg"
      }
    }
  }
}

// Room entry should look like:
{
  "title": "Deluxe Ocean View",
  "slug": "deluxe-ocean-view",
  "price": 299,
  "description": "Spacious room with ocean view...",
  "images": [
    {
      "fields": {
        "file": {
          "url": "//images.ctfassets.net/.../room1.jpg"
        }
      }
    }
  ]
}

Step 5: Test Your App

Run these commands:

cd /home/catwoman/beach-resort

Then visit: http://localhost:3000

Troubleshooting

If you see a blank page:

  • Open browser console (F12 → Console)
  • Check for errors
  • Look for the debug logs: console.log('Home data:', homeData)
  • Verify Contentful is returning data

Common Issues

  • No content in Contentful → Create entries
  • Wrong content type names → Use 'home' and 'room'
  • Images not uploaded → Upload to Media section
  • CORS errors → Check Contentful API settings
